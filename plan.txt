0. СИСТЕМНАЯ ОСНОВА
0.1. Создать persistent obj_game_controller:

Инициализация всех global. переменных (путь, уровень, карма и т.д.)

Переменная first_play — определяет, новая игра или нет.

0.2. Добавить obj_save_manager:

Загрузить/сохранить позицию игрока, комнату, прогресс, маршрут.

1. ГЛАВНОЕ МЕНЮ (rm_main_menu)
1.1. Нарисовать базовый интерфейс:

Кнопки: Start, Parameters, Quit.

1.2. Добавить ветвление Start:

Если first_play == true → новая игра сразу.

Если first_play == false → появляется подменю: Continue / Reset.

1.3. При выборе Reset:

Затемнение.

Появление Флауи (сцена).

Сброс глобальных данных.

2. НАЧАЛО ИГРЫ
2.1. Комнаты:

rm_intro_fall — падение в кусты.

rm_ruins_start — точка старта в руинах.

2.2. Логика переходов:

Новая игра → rm_intro_fall

Продолжить → загрузка комнаты + координат

После intro → переход в rm_ruins_start

3. ИГРОВАЯ СИСТЕМА (core gameplay)
3.1. obj_player:

Анимация в 4 направлениях.

Ходьба (стрелки), бег (X), взаимодействие (Z).

Поддержка смены спрайтов:

Условие	Спрайт
Пацифист / Нейтрал	spr_andy_default
Геноцид / Очищение	spr_andy_dusted

Подгружается при входе в комнату или после боёв/решений.

3.2. obj_interact_zone:

Сцена активации диалогов.

Разные типы: персонажи, объекты, указатели.

3.3. obj_dialogue_box:

Появляется снизу экрана.

Текст набирается посимвольно.

Управляется через Z/X.

3.4. obj_room_transition:

Анимация смены комнат + переход игрока.

4. СОХРАНЕНИЕ / ТОЧКИ СОХРАНЕНИЯ
4.1. obj_save_point:

Отображает диалог "Save / Cancel".

При выборе Save → сохраняет:

Координаты

Имя комнаты

Прогресс (karma, lv, route)

4.2. obj_save_manager:

Загружает всё выше при старте.

5. ПОДГОТОВКА К БОЕВОЙ СИСТЕМЕ
5.1. Структура боевого экрана:

Комната: rm_battle_base

Контроллер: obj_battle_controller

Интерфейс:

obj_UI_battle_box

Кнопки: def, act, item, spare

Магия (внутри def подменю)

5.2. Минимальный obj_enemy:

HP, имя, спрайт, поведение.

Реакция на act, spare.
